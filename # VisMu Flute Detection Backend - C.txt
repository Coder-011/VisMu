# VisMu Flute Detection Backend - Complete Build Instructions

**Project:** VisMu - Real-time Flute Finger Detection & Audio Synthesis System
**Frontend:** Ready-to-integrate UI with hand tracking visualization
**Task:** Build a full-stack backend that powers the VisMu interface

---

## PROJECT OVERVIEW

### What This App Does
1. **Captures** video from user's webcam showing hands playing a flute
2. **Detects** which of 6 fingers are pressing/lifting flute holes in real-time
3. **Maps** finger positions to Indian classical flute notes (Sa, Re, Ga, Ma, Pa, Dha, Ni)
4. **Synthesizes** corresponding flute audio in real-time using Web Audio API
5. **Animates** a synchronized 3D virtual flute showing finger positions
6. **Displays** confidence scores, latency metrics, and flute hole state visualization
7. **Brands** entire interface with "VisMu" logo and professional styling

### Key Metrics from UI
- **Latency Target:** 4.2ms or lower
- **Confidence Score:** 0.99+ (99% hand detection accuracy)
- **Hand Tracking:** Real-time MediaPipe pose estimation
- **Audio Latency:** Must be imperceptible (<50ms end-to-end)
- **Visualization:** 6 flute holes shown as cyan/dark circles
- **Active Sync:** 120 BPM Control Deck synchronization

## CORE BACKEND FEATURES

### 1. Hand Landmark Processing
**Input:** Array of 21 hand landmarks (x, y, z, confidence) from MediaPipe
**Process:**
- Extract fingertip positions (landmarks 4, 8, 12, 16, 20)
- Calculate finger lift/press state based on Y-axis position relative to palm
- Detect pressure via Z-axis depth values
- Smooth landmark data using Kalman filter (optional but recommended)
- Calculate confidence scores per finger

**Output:**
```javascript
{
  timestamp: 1234567890,
  fingers: {
    thumb: { lifted: false, confidence: 0.98, pressure: 75 },
    indexFinger: { lifted: true, confidence: 0.99, pressure: 0 },
    middleFinger: { lifted: true, confidence: 0.99, pressure: 0 },
    ringFinger: { lifted: false, confidence: 0.97, pressure: 68 },
    pinkyFinger: { lifted: false, confidence: 0.96, pressure: 72 },
    extraFinger: { lifted: false, confidence: 0.95, pressure: 70 }
  },
  overallConfidence: 0.992,
  latency: 4.2
}
```

### 2. Flute Hole Mapping Logic
**Flute Configuration:** Standard bansuri with 6 holes (H1-H6)

**Hole Numbering (Top to Bottom):**
- H1 = Top hole (closest to embouchure)
- H2 = Second from top
- H3 = Third from top
- H4 = Fourth hole (middle)
- H5 = Fifth hole
- H6 = Bottom hole

**Note Mapping Rules:**
```
Sa (440 Hz)  ← H1, H2, H3 closed (top 3) | H4, H5, H6 open
Re (494 Hz)  ← H1, H2 closed (top 2) | H3, H4, H5, H6 open
Ga (523 Hz)  ← H1 closed (top 1) | H2, H3, H4, H5, H6 open
Ma (587 Hz)  ← H1 open (all holes open)
Pa (659 Hz)  ← H1, H2, H3, H4, H5, H6 ALL CLOSED
Dha (739 Hz) ← H1, H2, H3, H4, H5 closed | H6 open (5 holes closed)
Ni (830 Hz)  ← H1, H2, H3, H4 closed | H5, H6 open (top 4 closed)
```

**Finger-to-Hole Mapping:**
```javascript
const fingerToHole = {
  thumb: 'H1',           // Top hole
  indexFinger: 'H2',     // Second hole
  middleFinger: 'H3',    // Third hole
  ringFinger: 'H4',      // Middle hole
  pinkyFinger: 'H5',     // Fifth hole
  extraFinger: 'H6'      // Bottom hole
};

// Note: "extraFinger" refers to the 6th detection point
// (could be palm, wrist, or secondary digit tracking)
```

**Hole State Logic:**
- **Closed:** Finger is lifted (Y-position above threshold) = hole is blocked
- **Open:** Finger is NOT lifted = hole is open
- **Threshold Tuning:** Calibrate Y-position threshold per user

### 3. Note Synthesis Engine
**Audio Source Strategy:**

**Option A: Web Audio API (Browser-side - PREFERRED)**
- Pre-load flute note samples (.wav/.mp3) for each note
- Trigger sample playback on note detection change
- Add crossfading between notes for smooth transitions
- Implement ADSR envelope (Attack, Decay, Sustain, Release)

**Option B: Server-side Audio Processing (Advanced)**
- Generate waveforms on backend using tone synthesis
- Stream audio via WebSocket or HTTP chunked responses
- Process with reverb/effects for realistic flute timbre

**Recommended:** Use **Option A** (pre-recorded samples) for lowest latency.

**Audio Files Needed:**
```
/audio/flute/
├── sa.wav        (440 Hz, 2-4 seconds, stereo, 16-bit)
├── re.wav        (494 Hz)
├── ga.wav        (523 Hz)
├── ma.wav        (587 Hz)
├── pa.wav        (659 Hz)
├── dha.wav       (739 Hz)
├── ni.wav        (830 Hz)
└── silence.wav   (100ms quiet, for transitions)
```

**Audio Playback Rules:**
1. Detect note change from finger state
2. If new note ≠ previous note, crossfade (50ms fade out old, fade in new)
3. If same note, continue playback
4. Implement note sustain (hold until finger lifts)
5. Add gentle envelope to prevent pops/clicks
6. Store audio context in backend for streaming if needed

### 4. Sensor Calibration System
**Why Calibration Matters:**
- Each user has different hand size, flute size, camera angle
- Lighting conditions affect MediaPipe confidence
- Need per-user Y-position thresholds for accurate hole detection

**Calibration Workflow:**
```javascript

POST /api/calibrate/record-position
// Input: landmark array, hole_index, action ("open" | "close")

// Calculates optimal thresholds per hole

```

**Calibration Data Structure:**

## DATABASE SCHEMA (SQLite Example)

```sql
-- Users & Sessions
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  lastActiveAt DATETIME,
  deviceInfo TEXT
);

CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  userId TEXT NOT NULL,
  startTime DATETIME DEFAULT CURRENT_TIMESTAMP,
  endTime DATETIME,
  noteCount INTEGER DEFAULT 0,
  accuracy REAL DEFAULT 0.0,
  FOREIGN KEY(userId) REFERENCES users(id)
);

-- Calibration Data
CREATE TABLE calibrationProfiles (
  id TEXT PRIMARY KEY,
  userId TEXT NOT NULL,
  sessionId TEXT,
  createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  holeThresholds TEXT, -- JSON string
  ambientLight TEXT,
  cameraModel TEXT,
  accuracy REAL,
  FOREIGN KEY(userId) REFERENCES users(id),
  FOREIGN KEY(sessionId) REFERENCES sessions(id)
);

-- Performance Metrics
CREATE TABLE detectionMetrics (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  sessionId TEXT NOT NULL,
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
  frameLatency REAL,
  confidence REAL,
  detectedNote TEXT,
  fingerState TEXT, -- JSON: { thumb: bool, ... }
  holeState TEXT,   -- JSON: { H1: bool, ... }
  FOREIGN KEY(sessionId) REFERENCES sessions(id)
);

CREATE TABLE audioEvents (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  sessionId TEXT NOT NULL,
  timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
  note TEXT,
  startTime REAL,
  duration REAL,
  confidence REAL,
  pressure REAL,
  FOREIGN KEY(sessionId) REFERENCES sessions(id)
);

-- Configuration
CREATE TABLE userPreferences (
  userId TEXT PRIMARY KEY,
  pressureSensitivity REAL DEFAULT 0.8,
  calibrationMode TEXT DEFAULT 'auto',
  audioFormat TEXT DEFAULT 'wav',
  smoothingAlgorithm TEXT DEFAULT 'kalman',
  updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY(userId) REFERENCES users(id)
);
```

## LANDMARK DATA FORMAT

### Input Structure (from MediaPipe Holistic)
```javascript
// 21 hand landmarks per hand
landmarks = [
  { x: 0.5, y: 0.3, z: 0.1, confidence: 0.99 },  // 0: Wrist
  { x: 0.52, y: 0.25, z: 0.12, confidence: 0.98 }, // 1: Thumb CMC
  { x: 0.54, y: 0.20, z: 0.15, confidence: 0.99 }, // 2: Thumb MCP
  { x: 0.56, y: 0.15, z: 0.18, confidence: 0.99 }, // 3: Thumb IP
  { x: 0.58, y: 0.10, z: 0.20, confidence: 0.99 }, // 4: Thumb TIP (fingertip)
  // ... (5-7: Index finger, 9-12: Middle, 13-16: Ring, 17-20: Pinky)
  // For flute detection, focus on fingertip landmarks: 4, 8, 12, 16, 20
];
```

### Y-Axis Interpretation for Hole Detection
```
Y = 0.0 (top of image, finger fully lifted)
Y = 0.5 (middle, neutral position)
Y = 1.0 (bottom of image, finger pressed down)

Convention: Lifted = Y < threshold (hole closed)
            Pressed = Y >= threshold (hole open)
```

### Confidence Filtering
- Reject landmarks with confidence < 0.7
- Flag warnings if overall hand confidence < 0.85
- Use previous frame data if confidence drop is sudden
